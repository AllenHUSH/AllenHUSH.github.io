# 关于MVC和MVVM的理解

# MVC

MVC是一种Web设计理念 M--Model模型 V--View视图 C--Controller控制器

# MVVM

其中 V是前端需要关注的地方 而我们又把V划分成MVVM

M--Model 保存每个页面中单独的数据 VM--ViewModel 调度者，分割了M和V，每当V层想到获取或保存数据的时候，都需要VM做中间处理 V--View 每个页面中的HTML结构

附图如下/源：黑马学院vue.js课程 以vue.js写一个HelloWorld为例，讲解哪些部分是M、VM、V

<body>
    <div class="app">
        <!-- 展现视图即V层 -->
        <p>{{msg}}</p>
    </div>
</body>
<script>
    var vm = new Vue({//new出来的vm即VM
        el: '.app',
        data: {//这里存储的数据就是M
            msg:'HelloWorld'
        }
    }) 
</script>
